Technical Blueprint

UniCore Platform

A Multi-Tenant Educational Infrastructure with Real-Time Academic Control

1. System Architecture

Multi-Database Tenancy

Unlike standard systems, UniCore uses a Database-per-School model. Each university has its own isolated MongoDB instance. This ensures maximum security, zero data leakage between schools, and independent scaling.

Real-Time Core

Powered by Socket.io, the system synchronizes attendance triggers, countdown timers, and notifications instantly across student and teacher portals without page refreshes.

Dynamic Context Switching

A custom middleware layer detects the school context from request headers, routing API calls to the correct database on the fly.

2. Core Modules & Functionality

🏫 Attendance Protocol (7-Minute Rule)

A high-security attendance system designed to prevent fraud. When a lecturer starts a session:

  • A unique 6-digit PIN is generated.
  • A 7-minute countdown begins (Real-time).
  • The PIN expires automatically after the window closes.
  • Students must enter the PIN while within the time-window.

🛡️ Leadership Protocol (Course Reps)

Course Representatives are democratically applied for and lecturer-verified:

  • Students apply for "Leadership Status" for specific units.
  • Requests appear in the Lecturer's "Rep Requests" portal.
  • Upon approval, the student gains administrative tools (PIN management).

🧠 AI Academic Assistant

Integrated throughout the student portal to provide research help, course summarization, and academic guidance based on university-specific materials.
