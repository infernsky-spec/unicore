Executive Summary

EduBridge is a comprehensive, cloud-ready university management system designed specifically for Ghanaian and African higher education institutions. It replaces fragmented paper-based systems and disconnected spreadsheets with a single, unified digital platform that every stakeholder — administrators, lecturers, students, and parents — can access from any device.

Unlike generic international platforms, EduBridge understands the Ghanaian university context: it supports all major universities from UG to KNUST to Central University, integrates with local fee payment methods like Mobile Money, and provides AI-powered academic assistance in a way that is relevant to students on the continent.

How the Attendance System Works

Real-Time PIN-Based Attendance

EduBridge uses a secure, real-time PIN system that ensures only physically present students mark attendance — eliminating proxy attendance fraud.

1

Lecturer Opens Session

The lecturer logs into EduBridge (web or mobile portal), selects their course, and creates an attendance session. A unique 6-digit PIN is generated instantly.

2

PIN Shared with Students

The lecturer displays the PIN on screen or reads it out. Students who are physically present can see it. The PIN is only valid for the duration set by the admin (default: 30 minutes).

3

Students Mark Attendance

Students log into the EduBridge portal on their phone, select their course, and enter the PIN. Students who mark within the first 15 minutes are "Present". After that, they are "Late".

4

Real-Time Dashboard

The lecturer sees a live feed of who has marked attendance. The count updates in real time via WebSocket technology.

5

Session Closed

When the lecturer closes the session, all students who have not marked are automatically recorded as "Absent". Attendance summaries update immediately.

6

Warnings Issued

The system automatically flags students below 60% attendance as "At Risk" and below 50% as "Critical". Both the student and their parent are notified.

Setup & Deployment

Local Development (VS Code)

Prerequisites

• Node.js v18 or higher — nodejs.org
• MongoDB Community Server — mongodb.com/try/download/community
• VS Code — code.visualstudio.com

1

Extract and Open

Extract the ZIP to your desktop. In VS Code: File → Open Folder → select the UniCore folder.

2

Install Dependencies

cd backend && npm install then cd ../frontend && npm install

3

Configure .env

The backend/.env file is pre-configured for local development. Change JWT_SECRET for production.

4

Seed the Database

cd backend && npm run seed — creates all test accounts.

5

Start Both Servers

Terminal 1: cd backend && npm run dev   Terminal 2: cd frontend && npm run dev

6

Open in Browser

Visit http://localhost:5173 — you will see the university selection screen.

🔐 Copyright & Intellectual Property Protection

EduBridge is a proprietary software product. To fully protect your intellectual property, follow these steps:

  1. Copyright Registration (Ghana): Register with the Copyright Office of Ghana (part of the Ghana Intellectual Property Office — GIPC). Visit gipc.gov.gh and file a "Copyright Registration Form" for software/digital works. Fee is minimal.
  2. Trademark the Name: File a trademark application for "EduBridge" and your company name "NexaVision Technologies" at GIPC. This protects the brand name and logo.
  3. Source Code Escrow: Archive a timestamped copy of your source code via a lawyer or notary public. This creates legal proof of creation date.
  4. Terms of Service: Add a Terms of Service and End-User Licence Agreement (EULA) to the platform stating that the software is proprietary and not open source.
  5. International Protection: For international markets, register under the Berne Convention (Ghana is a signatory) — this automatically protects your work in 180+ countries.
  6. WIPO Registration: Consider filing with the World Intellectual Property Organization (WIPO) at wipo.int for global software protection.
  7. Licence Model: Sell EduBridge as a SaaS (Software as a Service) with annual licensing fees per institution. Never give source code to clients — host it on your servers.
